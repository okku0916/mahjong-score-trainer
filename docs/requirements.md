# 麻雀点数計算練習アプリ 要件定義書

## 1. 文書情報

- プロジェクト名: mahjong-score-trainer
- ステータス: MVP要件確定
- 最終更新日: 2026-07-10

## 2. 目的

麻雀の符、翻数、最終点数をそれぞれ独立して反復練習できる、シンプルなWebアプリを提供する。利用者の習熟度は限定せず、練習モードによって学習対象を分ける。

## 3. 基本方針

- スマートフォンを優先したレスポンシブWebアプリとする。
- ユーザーが設定を行わず、すぐに練習を始められる構成とする。
- 難易度、問題数、制限時間は設けない。
- 問題は条件付きランダム生成とし、正解計算による検証を通過した問題だけを表示する。
- ユーザー登録、成績保存、総合問題はMVP対象外とする。

## 4. 対象範囲

### 4.1 MVP対象

- 符計算モード
- 翻数計算モード
- 点数計算モード
- 牌画像と局面情報の表示
- 選択式回答
- 即時採点
- 正解および計算内訳の解説
- 同一モードでの連続出題

### 4.2 MVP対象外

- ユーザー登録、ログイン
- 成績および学習履歴の永続化
- 難易度選択
- 問題数選択
- 制限時間、タイムアタック
- 手牌から役、符、翻、点数をすべて求める総合問題
- 三人麻雀

## 5. 画面構成

### 5.1 ホーム画面

以下の3モードだけを表示し、選択すると直ちに問題画面へ移動する。

- 符計算
- 翻数計算
- 点数計算

### 5.2 問題画面

1. 問題を表示する。
2. 選択肢から回答する。
3. 回答を確定し、正誤と解説を表示する。
4. 回答後は選択肢を変更できない。
5. 「次の問題」ボタンを押すと、同じモードの新しい問題を表示する。
6. ホームへ戻る操作を提供する。

問題数による終了画面は設けず、任意の回数だけ練習を継続できるようにする。

## 6. モード別要件

### 6.1 符計算モード

- 手牌、和了牌、門前・副露、ロン・ツモ、場風、自風など、符計算に必要な情報を表示する。
- 回答は符数の選択式とする。
- 選択肢は `20 / 25 / 30 / 40 / 50 / 60 / 70 / 80 / 90 / 100 / 110符` を基本とする。
- 採点後、面子、雀頭、待ち、和了方法などによる符の内訳を表示する。
- 役満の問題は出題しない。

### 6.2 翻数計算モード

- 手牌、和了牌、門前・副露、ロン・ツモ、場風、自風、ドラ表示牌などを表示する。
- 成立役の名称は回答させず、合計翻数だけを選択させる。
- 採点後、成立役、各役の翻数、ドラ・赤ドラの内訳を表示する。
- 役満、ダブル役満および役満の複合を出題対象とする。

### 6.3 点数計算モード

- 手牌画像は表示しない。
- 親または子、ロンまたはツモ、符、翻を問題条件として表示する。
- ロンでは受取点数、ツモでは各プレイヤーの支払点数を選択させる。
- 採点後、基本点と端数処理を含む計算結果を表示する。
- 満貫以上および複数倍役満を出題対象とする。

## 7. 出題・採点要件

- 出題テーマと局面条件を抽選した後、条件を満たす和了形を生成する。
- 生成後、役、符、翻、点数を計算エンジンで検証する。
- 役なし、不正な牌枚数、条件不一致などの問題は破棄して再生成する。
- 同一問題が連続しないようにする。
- 回答確定後、問題データと選択肢を変更しない。
- 「次の問題」の連打による重複生成を防止する。
- 問題生成ロジックと正解計算ロジックは可能な限り分離する。
- 代表的な固定問題による自動テストを用意する。

## 8. 採用ルール

- 一般的な四人麻雀
- 喰いタンあり
- 後付けあり
- 赤ドラあり（五萬、五筒、五索に各1枚）
- 切り上げ満貫なし
- ローカル役なし
- ダブル役満あり
- 役満の複合あり
- 数え役満は13翻以上を1倍役満とし、26翻以上でも倍増させない
- 連風牌の雀頭は2符
- 本場なし
- 供託なし
- 責任払い（包）なし
- フリテン判定なし
- 一発あり
- 裏ドラあり
- ダブル立直あり
- 海底撈月、河底撈魚、嶺上開花、槍槓あり
- 天和、地和あり

局進行に関係する役は、牌譜全体を再現せず、成立に必要な状況を問題画面のラベルとして明示する。

### 8.1 ダブル役満

- 国士無双十三面待ち
- 四暗刻単騎
- 純正九蓮宝燈
- 大四喜

別々の役満が同時成立した場合は複合させる。ダブル役満形と別の役満が複合した場合は3倍以上の役満も許可する。

### 8.2 点数例

切り上げ満貫を採用しないため、以下の点数とする。

- 子30符4翻ロン: 7,700点
- 親30符4翻ロン: 11,600点
- 子60符3翻ロン: 7,700点
- 親60符3翻ロン: 11,600点

## 9. 牌画像

- PNG形式の個別牌画像を使用する。
- 通常牌34種、赤牌3種、点棒4種を利用する。
- 大サイズ、縦向き、立体効果なしの画像を基本素材とする。
- 横向きの牌はCSSで回転して表示する。
- SVGへの変換は行わない。

### 9.1 素材情報

- 配布元: 麻雀豆腐
- 配布ページ: https://majandofu.com/mahjong-images
- 取得日: 2026-07-10
- 個人・商用利用可能
- 画像の加工可能
- 著作権は配布元に帰属
- リンク、ソーシャルブックマーク、出典表示は推奨

ホーム画面下部に「牌画像: 麻雀豆腐」と配布元へのリンクを表示する。

## 10. 技術要件

- React
- TypeScript
- Vite
- React Router
- CSS Modules
- Vitest
- React Testing Library
- Playwright
- ESLint
- Prettier
- 将来のPWA対応を考慮する

点数計算および問題生成のドメインロジックはReactから分離し、純粋なTypeScriptとして実装する。MVPではサーバーおよびデータベースを使用しない。

## 11. 将来要件

- モード別の回答数、正解数、正答率
- 役や符の論点別の正答率
- 苦手問題の優先出題
- 学習履歴
- 連続正解数
- ユーザー登録
- 複数端末間での成績同期
- 総合問題
- PWAおよびオフライン対応

将来の成績保存に備え、MVPの問題データは出題ID、モード、正解、ユーザー回答、論点タグを保持できる構造とする。

## 12. 問題生成失敗時の処理

- ランダム問題の生成と検証は、1問につき最大100回まで再試行する。
- 100回で有効な問題を生成できなかった場合は、検証済みの固定問題から出題する。
- 固定問題も利用できない場合に限り、エラー表示と再試行ボタンを表示する。
